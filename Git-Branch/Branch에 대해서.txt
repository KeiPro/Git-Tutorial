브랜치의 종류

1. 통합 브랜치

	: 언제든지 배포할 수 있는 버전을 만들 수 있어야 하는 브랜치.
	  이 브랜치는 늘 안정적인 상태가 중요하다. 현재 개발하고 있는 프로젝트가
	  해당하는 플랫폼에서 반드시 모든 기능이 정상적으로 동작되는 상태여야 한다. 
	  
	  만약 이 통합 브랜치에서 동작하는 기능에 버그가 발생하였을 경우나 새로운 기능을
	  추가해야된다면, 바로 토픽 브랜치를 만들어서 일을 처리해야 한다.
	  처음에는 보통 통합 브랜치에서 토픽 브랜치를 만들어낸다.
	  일반적으로 저장소를 처음 만들었을 때 생기는 master브랜치를 통합 브랜치로 사용한다.

2. 토픽 브랜치( = 피처 브랜치 )

	: 기능 추가나 버그 수정과 같은 단위 작업을 위한 브랜치이다. 여러 개의 작업을 동시에 진행할 때에는,
	  그 수만큼 토픽 브랜치를 생성할 수 있다. 토픽 브랜치는 보통 통합 브랜치에서 만들어 내며,
	  토픽 브랜치에서 특정 작업이 완료되면 다시 통합 브랜치에 머지하는 방식으로 진행된다.


	* ~(틸트), ^(캐럿)기호를 통해 'HEAD' 뒤에 붙여 몇 세대 앞의 커밋을 가리킬 수 있습니다.

	* stash : 커밋하지 않은 변경 내용이나 새롭게 추가한 파일이 인덱스와 작업 트리에 남아 있는 채로 다른 브랜치로
	           전환(checkout)하면 그 변경 내용을 전환된 브랜치에서 커밋할 수 있다.
		Stash란, 파일의 변경 내용을 일시적으로 기록해두는 영역.
